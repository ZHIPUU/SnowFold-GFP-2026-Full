## 二、关键修复：Fixed Position 1 (M)

### 2.1 Bug 发现 (R19 调试)

R19 评估时发现的严重 bug:
- R18/R19 MPNN 输出中 **97% 候选不以 M 开头**
- 实际只有 28/1000 候选通过 `seq.startswith("M")` 过滤
- **违反项目规则第一节: 首位必须为 M**

### 2.2 根因

```python
# 错误的 R18/R19 代码
FIXED = [65, 66, 67, 96, 222]  # 5 chromophore 残基
```

MPNN 默认不固定 position 1 (M), 必须**显式**告诉它。

### 2.3 修复

```python
# R20 修复
FIXED = [1, 65, 66, 67, 96, 222]  # 1 (M) + 5 chromophore
```

### 2.4 效果对比

| 指标 | R19 (无 pos 1) | R20 (有 pos 1) |
|:----:|:-------------:|:-------------:|
| 通过率 | ~8% | **62.8%** |

| Top 6 chromo | 0.94-0.95 | **0.957-0.965** |
| 6/6 候选来自 | 多父代 | R20 收敛到 R20 父代附近 |

**关键洞察**: "通过率"和"质量"双重提升 — Fixed position 1 让 MPNN 探索大幅扩展。
