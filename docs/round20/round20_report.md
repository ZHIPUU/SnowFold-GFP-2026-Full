## 二、方法论

### 2.1 关键修复：Fixed Position 1

R19 评估时发现一个严重 bug：

```python
# R18/R19 ❌ Buggy
FIXED = [65, 66, 67, 96, 222]  # 5 chromophore 残基
```

这导致 MPNN 把 position 1 (M) 重设计，**97% 候选不以 M 开头**。R20 修复：

```python
# R20 ✅ Fixed
FIXED = [1, 65, 66, 67, 96, 222]  # 1 (M) + 5 chromophore 核心
```

## 四、关键洞察

1. **Fixed position 1 (M) 是历史性 bug**
   - 原因：MPNN 默认不固定第一个残基，必须显式固定 [1]
   - 影响：之前 97% 候选不合规或分数被低估

2. **A800 算力 + 正确评分 = 突破性进步**
   - RTX 5080 16GB 只能本地跑小批量 (150 候选)
   - A800 80GB 让大规模搜索 (6000 候选) 成为可能

3. **生色团 pLDDT 0.95 → 0.96 的意义**
   - 之前被认为是"ESMFold 评分天花板"
   - 实际是 **MPNN 候选质量问题**（因为很多候选不以 M 开头）

4. **T=0.2 仍然是甜蜜点**
   - Top 6 的 6 条全部来自 T=0.2
   - 与 R18/R19 一致
